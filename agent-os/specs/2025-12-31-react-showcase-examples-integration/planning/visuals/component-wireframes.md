## Legend

```
Symbols used in wireframes:

[ ]     - Checkbox (unchecked)
[x]     - Checkbox (checked)
[o]     - Radio button (selected)
( )     - Radio button (unselected)
[v]     - Dropdown arrow (closed)
[^]     - Dropdown arrow (open)
[X]     - Close button
[!]     - Error/Warning icon
[~]     - Loading spinner
[ok]    - Success/Valid indicator
[...]   - More options menu
[+]     - Add/Create action
[cal]   - Calendar picker
[-->]   - Navigation arrow

Borders:
+---+   - Normal border
+===+   - Highlighted/focused border
+- -+   - Dashed border (dragging state)

Text styling:
~~text~~ - Strikethrough (completed items)
*text*   - Current/active item
```

## Responsive Considerations

```
MOBILE BREAKPOINT (< 640px):
- Modals become full-screen sheets
- Wizard steps stack vertically
- Form fields take full width
- Side-by-side fields stack

TABLET BREAKPOINT (640px - 1024px):
- Modals maintain max-width
- 2-column form layouts
- Wizard steps remain horizontal

DESKTOP BREAKPOINT (> 1024px):
- Full layouts as shown above
- Multi-column forms
- Side panels for quick actions
```
