## 4. Route State Machine Diagram

### 4.1 Main Navigation Flow Machine

```
                              +------------------+
                              |                  |
                              |   app.loading    |
                              |                  |
                              +--------+---------+
                                       |
                                       | [LOADED]
                                       v
                              +------------------+
           +----------------->|                  |<------------------+
           |                  |   app.ready      |                   |
           |                  |                  |                   |
           |                  +--------+---------+                   |
           |                           |                             |
           |          +----------------+----------------+            |
           |          |                |                |            |
           |          v                v                v            |
    +------+------+  +-----------+  +-----------+  +-----------+    |
    |             |  |           |  |           |  |           |    |
    | route.dash  |  | route.new |  | route.set |  | route.on  |    |
    |   board     |  |   Task    |  |   tings   |  |  boarding |    |
    |             |  |           |  |           |  |           |    |
    +------+------+  +-----+-----+  +-----+-----+  +-----+-----+    |
           |               |              |              |          |
           |               |              |              |          |
           +---------------+--------------+--------------+----------+
                                  |
                             [NAVIGATE]
                                  |
                                  v
                          Route Transition
```

### 4.2 Route Guards and Transitions

```
+=========================================================================+
|                     ROUTE STATE MACHINE                                 |
+=========================================================================+
|                                                                         |
|  STATES:                                                                |
|  -------                                                                |
|  - idle           : No navigation in progress                          |
|  - navigating     : Transition in progress                             |
|  - dashboard      : /                                                   |
|  - taskDetail     : /tasks/:id                                          |
|  - newTask        : /tasks/new                                          |
|  - settings       : /settings                                           |
|  - onboarding     : /onboarding (guarded)                               |
|                                                                         |
|  TRANSITIONS:                                                           |
|  ------------                                                           |
|                                                                         |
|  [idle] --NAVIGATE--> [navigating] --COMPLETE--> [target_route]         |
|                            |                                            |
|                            +--GUARD_FAIL--> [fallback_route]            |
|                            |                                            |
|                            +--ERROR--> [error_state]                    |
|                                                                         |
|  GUARDS:                                                                |
|  -------                                                                |
|  - onboarding: isNewUser === true                                       |
|  - taskDetail: task.exists === true                                     |
|  - settings: isAuthenticated === true                                   |
|                                                                         |
+=========================================================================+
```

### 4.3 Detailed Route Transitions

```
                                 START
                                   |
                                   v
                          +----------------+
                          |   app.init     |
                          +-------+--------+
                                  |
                       +----------+----------+
                       |                     |
                       v                     v
               +---------------+     +---------------+
               | isNewUser?    |     | isReturning?  |
               | YES           |     | YES           |
               +-------+-------+     +-------+-------+
                       |                     |
                       v                     v
              +----------------+    +----------------+
              | /onboarding    |    | /dashboard     |
              +----------------+    +----------------+
                       |
      +----------------+----------------+----------------+
      |                |                |                |
      v                v                v                v
  +-------+       +--------+      +---------+      +----------+
  | step1 |  -->  | step2  | -->  | step3   | -->  | complete |
  | profile|      | team   |      | prefs   |      |          |
  +-------+       +--------+      +---------+      +----+-----+
                                                        |
                                                        v
                                                 +-------------+
                                                 | /dashboard  |
                                                 +-------------+


ROUTE MAP:
==========

  /                    -->  dashboard.idle
  /tasks/new           -->  newTask.form
  /tasks/:id           -->  taskDetail.viewing
  /tasks/:id/edit      -->  taskDetail.editing
  /settings            -->  settings.general
  /settings/profile    -->  settings.profile
  /settings/theme      -->  settings.theme
  /onboarding          -->  onboarding.step1
  /onboarding/team     -->  onboarding.step2
  /onboarding/prefs    -->  onboarding.step3
```

### 4.4 Navigation Events and Side Effects

```
+-------------------------------------------------------------------------+
|  EVENT: NAVIGATE_TO_TASK_DETAIL                                         |
+-------------------------------------------------------------------------+
|                                                                         |
|  1. User clicks task in list                                            |
|       |                                                                 |
|       v                                                                 |
|  2. Dispatch: { type: 'NAVIGATE', payload: { route: '/tasks/:id' } }    |
|       |                                                                 |
|       v                                                                 |
|  3. Guard check: Does task exist?                                       |
|       |                                                                 |
|       +-- YES --> 4a. Update URL via React Router                       |
|       |                |                                                |
|       |                v                                                |
|       |           5a. Transition state to 'taskDetail.viewing'          |
|       |                |                                                |
|       |                v                                                |
|       |           6a. Effect: Fetch task data (React Query)             |
|       |                |                                                |
|       |                v                                                |
|       |           7a. Effect: Log navigation (Tracing)                  |
|       |                                                                 |
|       +-- NO --> 4b. Show error modal                                   |
|                      |                                                  |
|                      v                                                  |
|                  5b. Redirect to dashboard                              |
|                                                                         |
+-------------------------------------------------------------------------+
```

This wireframe document provides ASCII-based visual specifications for the TaskFlow Project Management Dashboard, covering:

1. **Application Layout** - Full responsive layouts for desktop, tablet, and mobile
2. **Dashboard View** - Task list, filters, stats cards, and empty states
3. **Navigation Sidebar** - Expanded, collapsed, and hover states
4. **Route State Machine** - Flow diagrams showing navigation state management
5. **Modal Patterns** - Confirmation, quick edit, and stacked modals
6. **Form Patterns** - Multi-step forms with validation states
7. **Wizard Flow** - Complete onboarding sequence
8. **DevTools Panel** - Container hierarchy and state inspection
9. **Settings View** - Nested navigation layout
10. **Responsive Breakpoints** - Layout adaptations for different screen sizes

These wireframes serve as the foundation for implementing the HexDI and HexDI Flow showcase application.
