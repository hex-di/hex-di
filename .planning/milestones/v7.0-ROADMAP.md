# Milestone v7.0: Distributed Tracing

**Status:** Shipped (2026-02-07)
**Phases:** 23-31
**Total Plans:** 37

## Overview

Replace the isolated per-container tracing system with distributed tracing that supports cross-container propagation, W3C Trace Context, multiple export backends (OpenTelemetry, Jaeger, Zipkin, DataDog), and framework integrations (Hono, React). The existing tracing system is removed entirely -- no compatibility shims, clean break.

## Key Decisions

- **Centralized Tree-Walking Subscription** for multi-container instrumentation (NOT tracer propagation through container hierarchy)
- **No changes to @hex-di/runtime hook inheritance** -- instrumentation is external, applied via hooks
- **No runtime package changes for MVP** -- @hex-di/tracing is a standalone package
- **@hex-di/tracing lives in packages/** (runtime library, not tooling)
- **Backend packages live in packages/** (@hex-di/tracing-otel, @hex-di/tracing-jaeger, etc.)
- **Module-level span stack** for active span context propagation across containers
- **WeakMap<InspectorAPI, Container>** for MVP reverse lookup (runtime change deferred to v8.0)
- **W3C Trace Context manual implementation** (no external dependency for header parsing)
- **OTel is the universal standard** -- Jaeger/Zipkin via OTel exporters; DataDog via dd-trace
- **Conditional hook registration** -- NoOp tracer results in zero hooks registered (0% overhead)
- **Object pooling not effective** for short-lived spans; modern JS engines optimize better
