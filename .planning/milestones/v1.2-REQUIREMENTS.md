## v1.2 Requirements

Requirements for Developer Experience milestone. Each maps to roadmap phases.

### Scoped Overrides

- [x] **SCOPE-01**: Container supports `withOverrides()` for temporary override contexts
- [x] **SCOPE-02**: `withOverrides()` accepts port-to-adapter map for type-safe overrides
- [x] **SCOPE-03**: Override context isolates instance memoization from parent container

### API Ergonomics

- [x] **API-01**: `defineService()` supports builder pattern with method chaining
- [x] **API-02**: Builder supports `.singleton()`, `.scoped()`, `.transient()` lifetime methods
- [x] **API-03**: Builder supports `.requires(...ports)` for dependency declaration
- [x] **API-04**: Builder supports `.factory(fn)` to complete service definition
- [x] **API-05**: `fromClass()` helper creates adapter from class constructor
- [x] **API-06**: `fromClass()` infers service type from class instance type

### Port System

- [x] **PORT-01**: `createInboundPort()` factory creates ports with `"inbound"` direction
- [x] **PORT-02**: `createOutboundPort()` factory creates ports with `"outbound"` direction
- [x] **PORT-03**: Port metadata supports description, category, and tags
- [x] **PORT-04**: `isDirectedPort()` type guard distinguishes directed from legacy ports
- [x] **PORT-05**: Directed ports maintain backward compatibility with existing `Port` type
