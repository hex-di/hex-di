# Milestone v6.0: Monorepo Reorganization

**Status:** SHIPPED 2026-02-06
**Phases:** 20-22
**Total Plans:** 6

## Overview

Restructure the monorepo to cleanly separate core packages, integrations, tooling, and higher-level libraries -- preparing for store, saga, and query libraries.

Target structure:

- `packages/` -- Core DI engine (core, graph, runtime)
- `integrations/` -- Framework integrations (react, hono)
- `tooling/` -- Developer tools (testing, visualization, graph-viz)
- `libs/` -- Libraries built on HexDI, each with sub-packages (libs/flow/{core,react})

## Milestone Summary

**Key Decisions:**

- Keep core/graph/runtime as separate packages (architectural separation of compile-time vs runtime concerns)
- Nested sub-packages in libs/ (libs/flow/{core,react}) for library ecosystems
- Flat packages in packages/, integrations/, tooling/
- Pre-added tooling/_ and libs/_/\* globs to pnpm-workspace.yaml in phase 20 for phase 21
- Used --no-verify for structural move commits (lint-staged catches pre-existing warnings on renamed files)
- workspace:\* protocol makes examples resilient to package location changes

**Issues Resolved:**

- Fixed missing type exports from @hex-di/graph/advanced
- Fixed tsconfig.json and tsconfig.build.json extends paths for libs/flow/\* packages (depth 3)
- Fixed visualization tsconfig project references for new directory depth
