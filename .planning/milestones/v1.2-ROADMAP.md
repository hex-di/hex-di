# Milestone v1.2: Developer Experience

**Status:** ✅ SHIPPED 2026-02-01

## Overview

Improve developer experience through better override patterns, ergonomic APIs, and enhanced port system clarity.

## Milestone Summary

**Key Decisions:**

| Decision                                            | Rationale                                       | Outcome |
| --------------------------------------------------- | ----------------------------------------------- | ------- |
| Factory-based overrides keyed by port name          | API simplicity, lazy instantiation              | ✓ Good  |
| Override context forks from parent singleton memo   | Inherit existing singletons, isolate new        | ✓ Good  |
| memoizeOwn() for override-specific caching          | Prevent overrides from inheriting cached values | ✓ Good  |
| Curried create<TService>()(name) for ServiceBuilder | Enables partial type application                | ✓ Good  |
| Immutable builder with Object.freeze                | Matches GraphBuilder pattern                    | ✓ Good  |
| DirectedPort as intersection with Port              | Backward compatible, gradual adoption           | ✓ Good  |

**Issues Resolved:**

- Override context returning cached singletons instead of mocks (fixed with memoizeOwn)
- Nested dependencies bypassing override context (fixed with resolveInternal check)
- createAdapter rejecting "request" lifetime (fixed by adding to VALID_LIFETIMES)

**Issues Deferred:**

- fromClass .as<Interface>('Name') type narrowing (TypeScript constraint limitation)

**Technical Debt Incurred:**

- fromClass interface narrowing workaround documented
