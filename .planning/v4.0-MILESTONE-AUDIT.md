### Phase 12: API Cleanup

Removed 4 deprecated methods:

- `provideAsync()` - replaced by unified `provide()` with auto-detect
- `provideUnchecked()` - removed, always use provide()
- `provideFirstError()` - removed, always use provide()
- `mergeWith()` - replaced by `merge()` with safe max depth

Renamed 1 method:

- `withUnsafeDepthOverride()` → `withExtendedDepth()`

All tests migrated to unified API.

### Phase 14: Bidirectional Captive Validation

**Key Finding:** The gap hypothesis was DISPROVEN.

Existing `FindReverseCaptiveDependency` already catches forward reference captive dependencies. When a singleton requiring unregistered scoped port is added first, then scoped adapter is added second, the type system produces an error.

Plan 14-02 (pending constraints with TPendingConstraints) was correctly skipped - unnecessary complexity avoided.
