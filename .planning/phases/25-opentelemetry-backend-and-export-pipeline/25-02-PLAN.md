        - Timeout after exportTimeoutMillis (default 30000) to prevent deadlock
    - After shutdown, onEnd becomes no-op (check isShutdown flag at start)
    - All async operations use console.error for logging — no external logger dependency

    - Internal state:
      - spanBuffer: SpanData[] for buffering
      - flushTimer: NodeJS.Timeout | undefined
      - isShutdown: boolean flag
    - Implement SpanProcessor interface:
      - onStart: no-op
      - onEnd:
        - Check if shutdown, return if true
        - If buffer >= maxQueueSize, drop oldest (shift)
        - Push spanData to buffer
        - If buffer >= maxExportBatchSize, flush immediately
        - Otherwise schedule flush with scheduledDelayMillis timeout
      - forceFlush:
        - Clear any pending timer
        - Export all buffered spans in batches of maxExportBatchSize
      - shutdown:
        - Set isShutdown flag
        - Clear pending timer
        - Flush all remaining spans
        - Call exporter.shutdown() with timeout (OTEL-08)
        - Use Promise.race with exportTimeoutMillis to prevent deadlock
    - Helper functions:
      - scheduleFlush(): Set timer if not already set
      - flush(): Export batch, handle errors gracefully
    - Ensure no memory leaks (clear timers, empty buffers)

  </action>
