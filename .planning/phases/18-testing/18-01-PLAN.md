## Goal

Create comprehensive test coverage for resolution hooks (`beforeResolve` and `afterResolve`) with 20+ scenarios covering basic operation, error handling, async resolution, scoped containers, and hook registration parity.

## Context

From research:

- Hooks are now built into Container via `addHook/removeHook` methods
- Can also be registered via `createContainer({ hooks: {...} })` options
- Hook context includes rich metadata: portName, lifetime, scopeId, isCacheHit, depth, containerId, etc.
- Existing async-resolution.test.ts has only 6 basic hook tests
- Must test both registration paths to ensure parity

From Phase 18 context:

- Test both error propagation AND cleanup guarantees when hooks throw
- Include scoped container interactions
- Parity tests: hooks added at runtime must behave identically to hooks at creation
- FIFO ordering is a documented contract
