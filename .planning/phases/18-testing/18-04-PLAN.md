## Goal

Create comprehensive test coverage for the TracingAPI with 16+ tests covering filtering, overhead measurement, cross-scope tracing, subscriptions, and statistics computation.

## Context

From research:

- TracingAPI exposed via `container.tracer` property (always available)
- TraceFilter supports 7 dimensions: portName, lifetime, isCacheHit, minDuration, maxDuration, scopeId, isPinned
- Methods: getTraces, getStats, pause/resume, clear, subscribe, pin/unpin
- Standalone functions: trace(), enableTracing()

From Phase 18 context:

- Full filter coverage: test each dimension independently
- Overhead test: verify disabled tracing has negligible cost
- Cross-scope: trace across parent/child boundaries
- Real-time subscriptions and stats computation
