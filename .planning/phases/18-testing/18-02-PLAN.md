## Goal

Create comprehensive tests for hook composition and ordering with 10+ scenarios covering FIFO ordering, lifecycle sequencing, mid-resolution removal, and cross-event interactions.

## Context

From research:

- Hooks support both beforeResolve and afterResolve events
- Multiple hooks can be registered for the same event
- FIFO ordering is a documented contract (registration order = execution order)
- removeHook can be called even during resolution

From Phase 18 context:

- Strict FIFO ordering guarantee must be tested
- Full lifecycle tests needed (both hooks on same resolution)
- Mid-resolution edge case: test removeHook during active resolution
- Hooks are read-only observers (cannot modify results)
