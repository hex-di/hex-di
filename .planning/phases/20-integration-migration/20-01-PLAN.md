<task type="auto">
  <name>Task 1: Move integration packages to new directory structure</name>
  <files>integrations/react/, integrations/hono/, pnpm-workspace.yaml</files>
  <action>
    1. Create the `integrations/` directory at the project root:
       ```bash
       mkdir -p integrations
       ```

    2. Move packages using `git mv` to preserve git history:
       ```bash
       git mv packages/react integrations/react
       git mv packages/hono integrations/hono
       ```

    3. Update `pnpm-workspace.yaml` to include the new workspace globs. Replace the current content with:
       ```yaml
       packages:
         - "packages/*"
         - "integrations/*"
         - "tooling/*"
         - "libs/*/*"
         - "examples/*"
         - "website"
       ```
       Note: `tooling/*` and `libs/*/*` are added now (for Phase 21) even though those directories don't exist yet. pnpm ignores globs that match nothing, so this is safe and avoids a second workspace config change later.

    4. Clean up stale node_modules and dist in moved packages:
       ```bash
       rm -rf integrations/react/node_modules integrations/react/dist
       rm -rf integrations/hono/node_modules integrations/hono/dist
       ```

    5. Run `pnpm install` to regenerate the lockfile with the new paths. This will re-resolve all `workspace:*` dependencies pointing to `@hex-di/react` and `@hex-di/hono` to their new locations automatically (pnpm resolves by package name, not path).

    6. Verify `packages/react` and `packages/hono` no longer exist.

  </action>
  <verify>
    - `ls integrations/react/package.json` succeeds
    - `ls integrations/hono/package.json` succeeds

    - `ls packages/hono 2>/dev/null` fails (directory gone)
    - `grep "integrations" pnpm-workspace.yaml` shows the new glob
    - `pnpm install` completes without errors
    - `pnpm ls @hex-di/react --json 2>/dev/null | grep -q "integrations"` confirms pnpm found it at new location
  </verify>
