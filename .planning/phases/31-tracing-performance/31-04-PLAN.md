    Create a generic object pool for reusable objects:
    - Generic `ObjectPool<T>` class with configurable max size
    - `acquire()` method that returns pooled object or creates new one
    - `release(obj: T)` method that returns object to pool (with reset callback)
    - `reset` callback to clean object state before reuse

  <files>packages/tracing/src/adapters/memory/tracer.ts, packages/tracing/src/adapters/memory/span.ts</files>
  <action>
    Modify Memory tracer to use object pooling for spans:
    - Create SpanPool using ObjectPool<MemorySpan>
    - Modify startSpan() to acquire span from pool instead of `new MemorySpan()`
    - Add reset() method to MemorySpan to clear state for reuse
    - Modify span.end() to release span back to pool after storing data
    - Ensure span data is cloned/copied before releasing span to pool
    - Keep circular buffer for span data storage (separate from span objects)

    Key insight: Pool the MemorySpan objects themselves, but still store the span data in the circular buffer. This reduces object allocation overhead while maintaining the same external API.

  </action>
