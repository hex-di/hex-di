## Goal

Implement React TracingProvider component and hooks (useTracer, useSpan, useTracedCallback) for first-class tracing support in React applications (FRMW-03 through FRMW-06).

## Tasks

<task>
Create TracingProvider component with React Context:
- Create TracingContext using React.createContext with null default
- Set displayName to "HexDI.TracingContext" for debugging
- Implement TracingProvider component accepting tracer and children props
- Follow existing ContainerProvider pattern for consistency
- Export TracingProvider from providers index
- Add @hex-di/tracing dependency to package.json
</task>

<task>
Implement useTracer hook for accessing tracer:
- Create useTracer hook that calls useContext(TracingContext)
- Throw descriptive error if used outside TracingProvider
- Follow existing usePort error handling pattern
- Return non-null Tracer instance
- Export from hooks index
</task>

<task>
Implement useSpan hook for getting active span:
- Create useSpan hook that uses useTracer
- Call tracer.getActiveSpan() to get current span
- Return Span | undefined for conditional logic
- No error if span is undefined (valid state)
- Export from hooks index
</task>

<task>
Implement useTracedCallback hook for tracing callbacks:
- Create useTracedCallback generic hook accepting name, callback, deps
- Use React.useCallback with tracer in dependency array
- Wrap callback execution in tracer.withSpan
- Preserve callback signature with proper typing
- Handle async callbacks correctly
- Export from hooks index
</task>

<task>
Write comprehensive tests for provider and hooks:
- Test TracingProvider provides tracer to children
- Test useTracer throws outside provider
- Test useSpan returns active span or undefined
- Test useTracedCallback creates spans for callbacks
- Use @testing-library/react for component testing
- Verify hooks follow React rules (no conditional calls)
- Test with createMemoryTracer for assertions
</task>
