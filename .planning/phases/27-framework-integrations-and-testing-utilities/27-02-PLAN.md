## Goal

Implement tracingMiddleware for Hono that extracts W3C Trace Context from requests, creates root spans, and injects context into responses (FRMW-01, FRMW-02).

## Tasks

<task>
Create TracingMiddlewareOptions interface and tracingMiddleware function:
- Define TracingMiddlewareOptions with tracer, spanName, extractContext, injectContext, attributes fields
- Implement tracingMiddleware factory function that returns MiddlewareHandler
- Default spanName to "${method} ${path}" pattern
- Default extractContext and injectContext to true
- Support custom attributes function for request-specific metadata
- Follow existing createScopeMiddleware pattern for consistency
</task>

<task>
Implement W3C Trace Context extraction and injection:
- Extract traceparent header from incoming request using extractTraceContext
- Create root span with parent context if available
- Set span kind to "server" and add HTTP semantic attributes
- Use try/catch/finally pattern to ensure span.end() always called
- Record exceptions and set error status on failure
- Inject trace context into response headers using injectTraceContext
</task>

<task>
Add Hono integration dependency on @hex-di/tracing:
- Update integrations/hono/package.json to add @hex-di/tracing as dependency
- Use workspace:* version for monorepo reference
- Update integrations/hono/src/index.ts to export tracingMiddleware
- Ensure proper module resolution with .js extensions
</task>

<task>
Write integration tests for tracing middleware:
- Test extraction of traceparent header from request
- Test span creation with correct name and attributes
- Test error handling and exception recording
- Test context injection into response headers
- Test optional extractContext/injectContext flags
- Test custom spanName and attributes functions
- Mock Hono context and verify middleware behavior
</task>
