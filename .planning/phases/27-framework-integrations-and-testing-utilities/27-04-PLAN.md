## Goal

Create performance benchmarks to verify NoOp tracer overhead < 5% and Memory tracer overhead < 10% compared to baseline (PERF-01, PERF-02).

## Tasks

<task>
Create baseline helper for consistent benchmark setup:
- Create baseline-helper.ts with test port and adapter definitions
- Define TestPort with simple object interface
- Create TestAdapter with transient lifetime (forces factory calls)
- Export createTestGraph() helper for consistent graph setup
- Use 100k resolution workload matching existing benchmarks
</task>

<task>
Implement NoOp tracer overhead benchmark:
- Create noop-overhead.bench.ts using vitest bench API
- Implement baseline benchmark with no instrumentation
- Implement NoOp benchmark with instrumentContainer(NOOP_TRACER)
- Use same 100k transient resolution workload for both
- Add descriptive benchmark names for clear reporting
- Call cleanup() after instrumented benchmark
</task>

<task>
Implement Memory tracer overhead benchmark:
- Create memory-overhead.bench.ts using vitest bench API
- Implement baseline benchmark with no instrumentation
- Implement Memory benchmark with instrumentContainer(createMemoryTracer())
- Use same 100k transient resolution workload
- Clear spans periodically to avoid memory growth affecting results
- Call cleanup() after instrumented benchmark
</task>

<task>
Configure vitest for benchmark execution:
- Update vitest.config.ts to include benchmark configuration
- Set appropriate warm-up and iteration counts
- Configure benchmark reporter for clear output
- Add npm script "test:bench" if not present
- Ensure benchmarks excluded from regular test runs
</task>

<task>
Verify performance requirements are met:
- Run benchmarks and capture results
- Calculate overhead percentage: (instrumented - baseline) / baseline * 100
- Verify NoOp overhead < 5% (PERF-01)
- Verify Memory overhead < 10% (PERF-02)
- Document results in code comments for future reference
</task>

- NoOp tracer overhead < 5% vs no tracing (PERF-01)
- Memory tracer overhead < 10% vs no tracing (PERF-02)
- Consistent 100k resolution workload for comparison
- Vitest bench configuration for statistical analysis
- Clear benchmark output showing ops/sec comparison
