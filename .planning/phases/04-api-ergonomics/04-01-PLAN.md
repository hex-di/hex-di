```typescript
requires<const TPorts extends readonly Port<unknown, string>[]>(
  ...ports: TPorts
```

- Uses rest parameters for ergonomic API: `.requires(PortA, PortB)`
- `const` modifier on TPorts preserves literal tuple type
- Returns NEW builder instance with TPorts as TRequires
- Replaces previous requires (not accumulates) - matches existing API semantics

**factory() terminal method:**

```typescript
factory(
  fn: (deps: ResolvedDeps<TupleToUnion<TRequires>>) => TService
): readonly [Port<TService, TName>, Adapter<...>]
```

Implementation:

1. Create port using `createPort<TName, TService>(this.name)`

   - `factory: fn`
3. Call `createAdapter(config)` to create the adapter
4. Return frozen tuple `[port, adapter]` using the existing `createServiceTuple` pattern

- Factory function `fn` receives correct deps type based on TRequires
- Return tuple has correct Port and Adapter types
- No type casting (per CLAUDE.md rules)
