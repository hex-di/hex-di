**ClassAdapterBuilder class:**

```typescript
class ClassAdapterBuilder<
  TInstance, // Class instance type
  TConstructorArgs extends readonly unknown[], // Constructor parameter types
> {
  private readonly classConstructor: new (...args: TConstructorArgs) => TInstance;

  private constructor(cls: new (...args: TConstructorArgs) => TInstance) {
    this.classConstructor = cls;
    Object.freeze(this);
  }

  static create<T extends new (...args: readonly unknown[]) => unknown>(
    cls: T
  ): ClassAdapterBuilder<InstanceType<T>, ConstructorParameters<T>>;

  /**
   * Names the service and optionally narrows the interface type.
   */
  as<TInterface extends TInstance = TInstance, const TName extends string = string>(
    name: TName
  ): ClassServiceBuilder<TInterface, TName, TConstructorArgs>;
}
```

**ClassServiceBuilder class:**

```typescript
class ClassServiceBuilder<
  TService,
  TName extends string,
  TConstructorArgs extends readonly unknown[],
  TRequires extends readonly Port<unknown, string>[] = readonly [],
  TLifetime extends Lifetime = "singleton",
> {
  // Similar to ServiceBuilder but:
  // - Stores class constructor instead of factory
  // - .requires() must match TConstructorArgs length and order

  singleton(): ClassServiceBuilder<...>;
  scoped(): ClassServiceBuilder<...>;
  transient(): ClassServiceBuilder<...>;

  requires<const TPorts extends readonly Port<unknown, string>[]>(
    ...ports: TPorts
  ): ClassServiceBuilder<TService, TName, TConstructorArgs, TPorts, TLifetime>;

  // Terminal - creates [Port, Adapter] tuple
  // NOTE: No factory() method - the class IS the factory
  build(): readonly [Port<TService, TName>, Adapter<...>];
}
```

**Implementation of build():**
Use the existing `createClassAdapter` from service.ts:

```typescript
build(): readonly [Port<TService, TName>, Adapter<...>] {
  const port = createPort<TName, TService>(this.name);
  const adapter = createClassAdapter({
    provides: port,
    requires: this.requires,
    lifetime: this.lifetime,
    class: this.classConstructor,
  });
  return Object.freeze([port, adapter]);
}
```

**fromClass() entry point:**

```typescript
export function fromClass<T extends new (...args: readonly unknown[]) => unknown>(
  cls: T
): ClassAdapterBuilder<InstanceType<T>, ConstructorParameters<T>> {
  return ClassAdapterBuilder.create(cls);
}
```

**Type safety for requires():**
The research notes that TypeScript cannot infer Port from constructor parameter type. Users MUST call .requires() with ports in the same order as constructor parameters. Document this clearly in JSDoc.
