Implement O(1) child container unregistration by replacing Array-based tracking with Map-based tracking.

Purpose: Current `unregisterChildContainer()` uses `Array.indexOf()` + `splice()` which is O(n). With many child containers, this becomes a bottleneck. Map-based tracking with numeric IDs achieves O(1) deletion.

Output: Updated `LifecycleManager` class with Map-based child container tracking while preserving LIFO disposal semantics.
</objective>

3. Replace `childContainers: Disposable[]` with:

   ```typescript
   private readonly childContainers: Map<number, Disposable> = new Map();
   private childIdCounter: number = 0;
   ```

4. Update `registerChildContainer()`:

   ```typescript
   registerChildContainer(child: Disposable): void {
     const id = this.childIdCounter++;
     child[CHILD_ID] = id;
     this.childContainers.set(id, child);
   }
   ```

5. Update `unregisterChildContainer()` to O(1):

   ```typescript
   unregisterChildContainer(child: Disposable): void {
     const id = child[CHILD_ID];
     if (id !== undefined) {
       this.childContainers.delete(id);
     }
   }
   ```

6. Update `dispose()` to maintain LIFO order:
   - Convert Map values to Array
   - Iterate in reverse order (last registered first)
   - Clear Map after disposal
