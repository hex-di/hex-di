       - Use parent inspector's getChildContainers() to find the new child
       - Match by childId from the event
       - For the matched child, use inspector.getContainer() to get HookableContainer
       - Fall back to iterating children if getContainer() not available

    2. Import childInspectorMap from lifecycle-manager:
       - Import the WeakMap exported from lifecycle-manager
       - Use it to retrieve child inspector by childId
       - This provides direct access to the child inspector

    3. Improve the listener logic:
       ```typescript
       const listener: InspectorListener = event => {
         if (event.type === "child-created") {
           // Get child inspector from the WeakMap using childId
           const { childInspectorMap } = await import("@hex-di/runtime/internal");
           const childInspector = childInspectorMap.get(event.childId);

           if (childInspector) {
             const childContainer = childInspector.getContainer?.() as HookableContainer;

             if (childContainer && !cleanups.has(childContainer)) {
               // Register mapping for this child's potential children
               registerContainerMapping(childInspector, childContainer);
               // Recursively instrument the new child
               walkTree(childContainer, childInspector);
             }
           }
         }
       };
       ```

    4. Optional: Add getContainerFromInspector() fallback:
       - First try inspector.getContainer?.()
       - Fall back to WeakMap lookup if method doesn't exist
       - This provides compatibility during migration

    Why this approach: Using getContainer() eliminates the reverse lookup timing issue. The container is immediately available when the event fires.
