       - After adding child to childContainers Map
       - Get the parent container's wrapper from this.parentState
       - Access parent inspector via parentState.wrapper?.inspector
       - If parent inspector exists and has emit method:

           ```typescript

             type: "child-created",
             childId: childState.id,
             childKind: childKind

           ```
       - Store child inspector reference in WeakMap for tree.ts to retrieve:
           ```typescript
           // At module level: export const childInspectorMap = new WeakMap()

           ```

    2. Update callers to pass childKind:
       - createChild: pass "child" to registerChildContainer
       - createChildAsync: pass "child" to registerChildContainer
       - createLazyChild: pass "lazy" to registerChildContainer
       - Detect correct childKind from creation method

    3. Ensure timing is correct:
       - Event must be emitted AFTER child is fully constructed
       - Event must be emitted AFTER child has inspector attached
       - Event must be emitted AFTER child is in parent's childContainers Map
       - This ensures tree.ts can successfully instrument when it receives event

    Why emit from LifecycleManager: This is the central place where ALL child containers are registered, regardless of creation method (sync/async/lazy). The childInspectorMap allows tree.ts to retrieve the child inspector using the childId from the event.
