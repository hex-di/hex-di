    Modify LifecycleManager.registerChildContainer() to emit child-created events:

    1. Update registerChildContainer() method:
       - After adding child to childContainers Map
       - Get the parent container's inspector via parentState.wrapper?.inspector
       - If parent inspector exists and has emit method:
         - Get child inspector from child.inspector property
         - Emit child-created event with metadata:
           ```typescript
           parentInspector.emit({
             type: "child-created",
             timestamp: Date.now(),
             metadata: {
               childId: childState.id,
               childName: child.name || "unnamed",
               childInspector: child.inspector
             }
           });
           ```

    2. Handle all container creation paths:
       - createChild: already calls registerChildContainer via onWrapperSet
       - createChildAsync: same flow
       - createLazyChild: same flow
       - All go through the same registration, so one change covers all

    3. Ensure timing is correct:
       - Event must be emitted AFTER child is fully constructed
       - Event must be emitted AFTER child has inspector attached
       - Event must be emitted AFTER child is in parent's childContainers Map
       - This ensures tree.ts can successfully instrument when it receives event

    Why emit from LifecycleManager: This is the central place where ALL child containers are registered, regardless of creation method (sync/async/lazy).
