<feature>
  <name>Circular Dependency Detection</name>
  <files>packages/graph/src/validation/circular-detection.ts, packages/graph/tests/validation/circular-detection.test.ts</files>
  <behavior>
    Detect circular dependencies at compile time using type-level DFS:

    Cases:
    - Simple cycle (A -> B -> A): Returns CircularDependencyError<['A', 'B', 'A']>
    - Complex cycle (A -> B -> C -> A): Returns CircularDependencyError<['A', 'B', 'C', 'A']>
    - No cycle (A -> B -> C): Returns never (no error)
    - Self reference (A -> A): Returns CircularDependencyError<['A', 'A']>
    - Multiple paths, one circular: Detects the cycle
    - Depth limit exceeded: Returns DepthExceededWarning type

  </behavior>
  <implementation>
Use recursive conditional types with visited set tracking:

1. DetectCircularDependency type:
   - Takes dependency graph, starting node, visited set, path, depth
   - Checks if current node in visited set (cycle found)
   - If cycle: return CircularDependencyError with path
   - If not: recurse on dependencies with updated visited/path
   - Track depth to prevent infinite recursion

2. CircularDependencyError type:
   - Template literal with full path
   - Shows A -> B -> C -> A format
   - Actionable fix suggestions

3. Integration with GraphBuilder validation pipeline
   </implementation>
   </feature>
