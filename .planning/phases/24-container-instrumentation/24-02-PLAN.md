    Implementation requirements:

    1. Track cleanups with Map&lt;Container, () => void&gt; for all instrumented containers

    2. Create internal `instrumentOne` function that:
       - Checks if container already in cleanups Map (skip if yes)
       - Calls instrumentContainer with tracer and options
       - Stores cleanup in Map

    3. Create `walkTree` function that:
       - Instruments the container via instrumentOne
       - Gets child containers via inspector.getChildContainers()
       - For each child InspectorAPI:
         * Try to get Container via getContainerFromInspector
         * If found, recursively walk it
       - Handles INST-09: parent container span is parent of child resolution spans (via shared span stack)

    4. Main function flow:
       - Walk existing tree starting from root
       - Subscribe to inspector events for new containers (per user decision on live subscription)
       - Listen for 'child-created' or similar events
       - Auto-instrument new children as they're added

    5. Return cleanup function that:
       - Unsubscribes from inspector events
       - Calls all cleanup functions in the Map
       - Clears the Map
       - Is idempotent (safe to call multiple times)

    Import instrumentContainer from ./container.js and utils from ./utils.js
    Handle missing container gracefully (some InspectorAPIs may not map to containers yet)
