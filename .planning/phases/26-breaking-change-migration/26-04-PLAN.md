<task type="auto">
  <name>Task 2: Migrate examples to new tracing API</name>
  <files>examples/</files>
  <action>
    For each example using old tracing, apply this migration pattern:

    1. Replace old imports:
       OLD: import { MemoryCollector } from "@hex-di/runtime";
       NEW: import { MemoryTracerAdapter, TracerPort, instrumentContainer } from "@hex-di/tracing";

    2. Replace trace/enableTracing pattern:
       OLD:
       ```typescript
       import { trace, enableTracing } from "@hex-di/runtime";
       enableTracing(container, new MemoryCollector());
       const result = trace(container, "operation", () => { ... });
       ```

       NEW:
       ```typescript
       import { instrumentContainer, TracerPort, MemoryTracerAdapter } from "@hex-di/tracing";
       // Add tracer to graph
       const graph = GraphBuilder.create()
         .provide(MemoryTracerAdapter)
         .build();
       const container = createContainer({ graph });
       // Instrument the container
       instrumentContainer(container, container.resolve(TracerPort), {
         portFilter: { include: ["*"] }
       });
       // Operations are traced automatically
       const result = container.resolve(SomePort);
       ```

    3. Remove container.tracer usage:
       OLD: const spans = container.tracer.getSpans();
       NEW: const tracer = container.resolve(TracerPort);
              const spans = (tracer as MemoryTracerAdapter).getSpans();

    4. Update each example's package.json if needed:
       - Add "@hex-di/tracing" to dependencies if using tracing
       - Remove any direct dependency on old collector types

    Apply systematically to each example identified in Task 1.

  </action>
  <verify>! grep -r "container\.tracer\|trace(\|enableTracing(" examples/ --include="*.ts" --include="*.tsx"</verify>
  <done>All examples migrated to new tracing API</done>
</task>
