## Tech Debt Summary

### Phase 24: Container Instrumentation (3 items)

1. **Dynamic child auto-instrumentation non-functional** — `instrumentContainerTree` walks existing hierarchy correctly, but live subscription for dynamically created children fails because: (a) runtime doesn't emit `child-created` events, (b) InspectorAPI-to-Container reverse lookup has chicken-and-egg problem. Already tracked as v8.0 deferred requirement ENH-05.

2. **Zero behavioral tests for instrumentation** — All 223 tracing tests pre-date or post-date Phase 24's instrumentation code. No unit tests for `instrumentContainer`, `createTracingHook`, span stack LIFO ordering, port filtering, cleanup functions, error recording, or parent-child span relationships. Instrumentation is tested indirectly via react-showcase example tests (tracing.test.ts:34-127), but no dedicated unit test suite exists.

3. **Inspector-to-Container reverse lookup design gap** — `tree.ts` uses a WeakMap-based `registerContainerMapping`/`getContainerFromInspector` pattern that cannot look up newly created child containers because registration happens during tree walk, not container creation. Tracked under ENH-04 in v8.0 requirements.

### Phase 25: OTel Export Pipeline (1 item)

4. **No automated tests for OTel packages** — `packages/tracing-otel/`, `packages/tracing-jaeger/`, `packages/tracing-zipkin/`, and `packages/tracing-datadog/` have no test directories. All components verified by code inspection, typecheck, and build, but zero automated regression tests exist.

### Phase 27: Framework Integrations (4 items)

5. **NoOp tracer overhead ~38%** — Target was <5%. Overhead comes from hook invocation machinery (beforeResolve/afterResolve function calls), not the NoOp tracer itself. Documented in benchmark.

6. **Memory tracer overhead ~602%** — Target was <10%. Overhead comes from span creation, serialization, and storage. Documented in benchmark. Expected for full in-memory tracing.

7. **Pre-existing lint warnings in type-guards.ts** — 2 `@typescript-eslint/no-unsafe-call` warnings from dynamically accessed crypto/console APIs. Intentional pattern.

8. **Hono middleware lint warnings** — 4 warnings from Hono Headers API typing limitations. Not actual unsafe code.

### Total: 8 items across 3 phases
