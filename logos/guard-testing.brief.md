# @hex-di/guard-testing

> See [LOGO-BRIEF.md](./LOGO-BRIEF.md) for shared brand context, color family, and technical specs.

## What It Is

Testing utilities for guard: memory adapters (in-memory audit trail, mock subject provider, mock signature service), subject fixtures, policy testing builders, conformance suites.

## Role & Metaphor

**The Security Test Range.** A controlled shooting range where guard policies are tested, subjects are simulated, and audit trails are verified - all in-memory, all disposable, all isolated.

## Key Concepts to Convey

- Testing guard policies in isolation
- Memory/mock implementations
- Conformance verification
- Controlled security testing

## Visual Direction

- The guard shield-hex but with dashed outlines (test boundary, like `@hex-di/testing`)
- A crosshair or target overlay on the shield (testing/targeting policies)
- Should reference both guard (shield) and testing (dashed outline)

## Signature Colors

- Base: `#DC2626` (red) from guard
- Testing overlay: `#EAB308` (amber) for the dashed frame

## Designer Prompt

> Design a minimal geometric logo icon (200x200 SVG) combining the guard shield-hexagon motif (red #DC2626, stroke only) with a testing indicator: the shield outline uses dashed strokes (12px dash, 6px gap). Inside, place a crosshair/target symbol (two thin perpendicular lines with a small circle at center) in amber (#EAB308). Conveys: "testing and targeting guard policies in a controlled environment." No text. Dark background friendly.
