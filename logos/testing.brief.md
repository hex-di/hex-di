# @hex-di/testing

> See [LOGO-BRIEF.md](./LOGO-BRIEF.md) for shared brand context, color family, and technical specs.

## What It Is

Test utilities: override adapters, create mocks, manage test container lifecycles, graph assertions, snapshot testing, React test rendering with DI.

## Role & Metaphor

**The Test Laboratory.** A controlled environment where you can swap real services for doubles, isolate components, and verify behavior - like a clean room for dependency injection testing.

## Key Concepts to Convey

- Testing: verification, controlled environment
- Mock/Override: replacing real with fake
- Isolation: separate test from production
- Swap: one adapter replaced by another

## Visual Direction

- A hexagon with a dashed/dotted outline (indicating "mock" or "test double" - not the real thing)
- Inside: two overlapping elements where one is swapping out the other (perhaps a solid circle being replaced by an outlined circle)
- Or: a hexagon with one edge highlighted/different from the others (the overridden adapter)

## Signature Color

`#EAB308` (amber/yellow) - testing, caution, laboratory

## Designer Prompt

> Design a minimal geometric logo icon (200x200 SVG) of a hexagon with dashed outline strokes (dash pattern: 12px dash, 6px gap) in amber (#EAB308, stroke 4px). Inside, show a swap/replacement motif: a solid filled circle on the left with an arrow pointing right toward a circle with only an outline (same color, no fill). The arrow is a simple chevron. Conveys: "a test environment where real services are swapped for mocks." The dashed hexagon signals "this is not production - it's a test boundary." No text. Dark background friendly.
